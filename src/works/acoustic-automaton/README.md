# 音響的オートマトン / Acoustic Automaton

## フォルダ構成

### 🎭 **作品固有実装** (`src/works/acoustic-automaton/`)

```
src/works/acoustic-automaton/
├── performance/                 # パフォーマンス制御システム
│   ├── acousticAutomaton.ts    # 音響処理（第1部）
│   ├── visualSystem.ts         # 映像システム（第1部）
│   └── performanceController.ts # 統合制御システム
├── ui/                         # 作品専用UI
│   └── performance.html        # パフォーマンス実行画面
├── audio/                      # 作品固有の音響処理
│   └── routing/               
└── visual/                     # 作品固有の映像処理
```

### 🔧 **共通エンジン** (`src/engine/`)

```
src/engine/
├── audio/                      # 音響エンジン
│   ├── core/                   # 基本音響機能
│   ├── devices/                # デバイス管理
│   ├── processing/             # 音響処理
│   └── dsp/                    # DSP統合
├── visual/                     # 映像エンジン
├── framework/                  # フレームワーク基盤
└── timing/                     # タイミング制御
```

## 技術仕様

### 音響
- **Web Audio API**: 低遅延音響処理
- **Channel Splitting**: 32チャンネル個別制御対応
- **Pitch Detection**: リアルタイム音程解析
- **Reverb Processing**: インパルスレスポンス生成

### 映像
- **Canvas 2D**: 高フレームレート描画
- **3分割レイアウト**: 楽器対応表示
- **リアルタイムエフェクト**: フラッシュ・軸線・減衰

### 制御
- **セクション管理**: 時間制限付き自動進行
- **デバイス統合**: 既存InputManager連携
- **手動テスト**: デバッグ用トリガー機能

## パフォーマンス設定

### 対象音程
- **B4 (493.88 Hz)**: 基準音程
- **検出閾値**: -30dB（調整可能）
- **音響インスタンス**: 最大20個保持

### 映像設定
- **解像度**: 1920x1080推奨
- **フレームレート**: 60FPS
- **フラッシュ時間**: 2秒減衰
- **軸線時間**: 3秒減衰

### セクション時間
- **テストモード**: 30秒
- **第1部**: 3分
- **第2部以降**: 未実装

## 使用方法

### 1. システム初期化
1. パフォーマンス画面を開く
2. 「システム初期化」をクリック
3. マイク接続状態を確認

### 2. テスト実行
1. 「テスト開始 (30秒)」をクリック
2. 手動トリガーで動作確認
3. B4音程で自動検出テスト

### 3. 本番パフォーマンス
1. 「第1部開始 (3分)」をクリック
2. 楽器演奏で自動進行
3. 制御パネルで監視
